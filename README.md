# L9_Framework: Custom Gym API 🏋️‍♂️

> **Важно:** В проекте не используются сторонние фреймворки (Express, Koa, Nest). Вся логика маршрутизации, middleware и обработки запросов написана с использованием стандартных модулей Node.js.

## 🛠 Технологический стек
* **Core:** Node.js HTTP Module
* **Routing:** EventEmitter (для создания системы событийных маршрутов)
* **Streams:** Обработка входящего потока данных (Request Body)
* **Environment:** dotenv

## 🚀 Основные возможности
* **Архитектура Middleware:** Поддержка цепочки промежуточных обработчиков через метод `.use()`.
* **REST Routing:** Полная поддержка методов `GET`, `POST`, `PUT`, `DELETE`.
* **Расширенные объекты req/res:** * `req.body` — автоматически распарсенный JSON.
  * `req.query` — параметры URL.
  * `res.json()` — метод для отправки JSON-ответов.
  * `res.status()` — удобная установка статус-кодов.
* **Error Handling:** Глобальный перехват ошибок (try/catch) внутри цикла обработки запроса, предотвращающий падение сервера.

## 📂 Структура проекта
* `Application.js` — ядро фреймворка, логика сервера и роутинга.
* `main.js` — точка входа, описание маршрутов и бизнес-логика "Тренажерного зала".
* `.env` — конфигурация порта сервера.

## 🧪 Примеры запросов (Postman)
Для тестирования сущности Gym Member используются следующие эндпоинты:
    
    GET /members — получить всех участников зала.
    POST /members — зарегистрировать нового участника.
    PUT /members?id=123 — обновить данные участника.
    DELETE /members?id=123 — удалить участника.
